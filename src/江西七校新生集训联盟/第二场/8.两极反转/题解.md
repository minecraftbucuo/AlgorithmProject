# 两级反转

首先,为了减少我们的分析难度,我们可以尝试考虑在只有操作2的情况下是否能使两个数$x,y$相同,同时也是为了方便将32bit降低到8bit考虑

思考一下,我们是否可以在只操作数x的情况下,将x变成y  
x = 0010 0100
y1 = 0000 0010
y2 = 0100 1010
需要发现的是,将连续的1视作一体后,操作2不可能将这两个连续的1变成1个连续的1,也不可能变成3个,这是因为操作2的条件要求修改的那一位前后不同,但是我们可以任意延长或缩短这连续的1形成的区域块

观察到这一点之后,答案就比较明显了   
x能够变成y的是,需要完全相同的0,1块  
例如0011 0011 和 0101 111 ,将连续的0和连续1视为一体后    
是同样的0101 的格式

然后我们再来处理可以取反所有位的操作  
明显的是,块0101 是可以变成1010的,意味着只要这样分块看后,块的个数相同,那么就一定可以变成相同的数  

再来处理区间询问的问题  
我这里直接使用x=y来表示x可以变成y  

可以发现 成为好数组的条件是 $x_1=x_2=x_3=..$   
只要区间内出现一个$\ne$那么就不可能成为好的数组

所以可以利用等式的传递性

用$e_i$来表示$arr_{i-1}$是否$=arr_i$  
我用$e_i=0$视为$=$,否则视为$\ne$  

只需要检验$e$数组区间内是否存在1即可   
可以利用前缀和检验
