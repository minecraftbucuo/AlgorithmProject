## A - 999

我们先来看一个低级点的问题

读者肯定知道一个数能被 $3$ 整除的条件,是去看这个数的各个数位的累加和是否是 $3$ 的倍数,读者可以简单自行证明一下

这里降低了难度只是询问了是否是 $9$ 的倍数,是为了让大家即使没见过也能猜出答案,让大家敢于猜想,再去证明其数学上或者逻辑上的正确性,严谨性

类似的,我们猜想一个数能被 $9$ 整除的条件是,这个数的各数位的和能被 $9$ 整除。

证明如下：  
假设 $x =a_n\cdots a_2a_1a_0$ ,其中 $a_i$ 表示这个数的第 $i$ 个数位的值
那么 $x =\Sigma^{n}_{i=0} a_i \times 10^i$  
我们要去验证这个式子对 $9$ 取余的结果是否为 $0$  

事实上如果有见过要求一些因为答案过大而要求你对答案对一个数 $p$ 取模的题时,你会没有多想的去用上这么一个东西

### **同余原理**   

1. $(a+b) \%p= (a\%p+b\%p)\%p$
2. $(a\times b) \%p= ((a\%p) \times (b\%p))\%p$  

这两个等式能成为我们计算中途就自信取余的底气,这两个等式读者自证不难,可自行尝试证明

故 $(\Sigma^{n}_{i=0} a_i \times 10^i)\%9=(\Sigma^{n}_{i=0} (a_i\%9) \times (10^i\%9))\%9=(\Sigma^{n}_{i=0} a_i)\%9$  

计算验证即可

### 参考代码
~~~c++
#include<iostream>
using namespace std;
int main()
{
    int T;
    cin>>T;
    while (T--)
    {
        string str;
        cin>>str;
        int r=0;
        for (int i=0;i<str.size();i++)r=(r+(str[i]-'0'))%9;
        if (r)cout<<0<<endl;
        else cout<<1<<endl;
    }
    return 0;
}
~~~

## B - 冒险

分类讨论即可

先讨论明显不可行的方案

$n>m$ 时,显然不可以,通过操作 $n$ 只会变大,不可能到达 $m$

$n+1=m$ 显然不可以,一次操作至少增加 $2$ ,不可到达 $m$

讨论可行的方案
显然在 $n<=m \space\&\& \space m \ne n+1$ 的情况一定有解
$n=m$ 的情况答案为 $0$

那么对于其他情况  
设 $d=m-n$ 则 $d>=2$  
实际上我们是去看是否只用 $2,3$ 两种数字拼出 $d$ ,即 $k_1*2+k_2*3 =d$
我们的答案是 $k_1+k_2$ 的最小值  
我们贪心地想尽可能的多用大的数 $3$ ,来使答案更小

对此,我们对 $d$ 模 $3$ 的值进行分类讨论：

1. $d\%3=0$ ：此时$k_1=0,k_2=\frac{d}{3}$ 显然最优
2. $d\%3=2$ ：此时 $k_1=1,k_2=\lfloor\frac{d}{3}\rfloor$ 显然此时答案是最优,如果不能发现,可以尝试将 $2$ 个 $3$ 替换成 $3$ 个 $2$ (保证和还是 $d$ ),可以发现答案一定会更差
3. $d\%3=1$ ：此时 $k_1=2,k_2=\lfloor\frac{d}{3}\rfloor-1$ 验证方式最优方式同上

### 参考代码

```C++
#include<bits/stdc++.h>
using namespace std;

int main()
{
    int T;cin>>T;
    while (T--)
    {
        int n,m;cin>>n>>m;
        if (m<n||m==n+1)cout<<"QAQ"<<endl;
        else
        {
            int d=m-n;
            int ans;
            if (d%3==0)ans=d/3;
            else ans=d/3+1;
            cout<<ans<<endl;
        }
    }
    return 0;
}
```

## C - 贪婪鼠鼠

需要注意到小橘猫的要回食物的规则为
鼠鼠取得的食物相同返回的是烧鸡
取得的食物不同返回的是哦润吉
对应着异或运算的定义

而异或运算可以视为一种不进位的加法 
这意味着异或运算是满足结合律的 
即 a ^ b ^ c = a ^ ( b ^ c )
这也导致鼠鼠取出食物的顺序是不会影响最后结果的

答案即 $n$ 个 $1$ , $m$ 个 $0$ 异或的结果

### 参考代码

~~~c++
#include<iostream>
using namespace std;

int main()
{
    int T;
    cin>>T;
    while (T--)
    {
        int n,m;cin>>n>>m;
        if (n&1)cout<<"1.00"<<endl;
        else cout<<"0.00"<<endl;
    }
    return 0;
}
~~~

## D - 签到题

题目只有 $32$MB 的空间限制,无法用常规方法通过此题。由上一题题解的启发，可以用异或运算进行求解。考虑到异或运算具有结合性，那么先将所有出现偶数次的数字进行异或，结果为 $0$ , 而唯一一个出现奇数次的数字全部异或，得到它本身。由此，全部数字进行异或的结果就是答案。

### 参考代码

```C++
#include <cstdio>
using namespace std;

signed main() {
	int n;
	scanf("%d", &n);
	long long ans = 0, x;
	while (n--) {
		scanf("%lld", &x);
		ans ^= x;
	}
	printf("%lld", ans);
	return 0;
}
```

## E - 3n+1

考虑逆操作：

1. 如果 $m$ 为偶数,则将 $m$ 除以 $2$ 。
2. 将 $m$ 乘 $3$ 加 $1$ 。

容易想到，如果 $n$ 能通过正操作变成 $m$ , 那么 $m$ 通过逆操作就一定能变成 $n$ , 反之同理。

我们考虑对 $m$ 逆操作,并且添加限制：当 $m$ 为偶数时令 $m = m / 2$ , 当 $m$ 为奇数时令 $m = 3 \times m + 1$ ，此时这就是个“3n+1猜想”,即任意正整数都能在此规则下变成 $1$ 。虽然是个猜想,但是在给定数据范围内的数字都已经得到了验证,所以只需要输出 “$1$” 即可。

## F - 111

假设 $f(n)$ 为长度为 $n$ 的满足条件的 “$01$字符串” (下面简称“串”)的个数。考虑求解其递推式：

1. 当第 $n$ 个数为 $0$ 时：

   对于前面的长度为 $n - 1$ 的串,每一种合法情况再加上第 $n$ 个数的 $0$ , 结果依然是合法的,此时有 $f(n - 1)$ 种合法的串。

2. 当第 $n$ 个数为 $1$ 时：

   发现不像情况 $1$ 一样那么简单,因为对于每一种合法的长度为 $n - 1$ 的串,在其结尾加上 “$1$” 并不能保证结果合法,此时需要对第 $n - 1$ 个数再讨论：

   1. 当第 $n - 1$ 个数为 $0$ 时：

      显然合法串数量为 $f(n - 2)$ 。

   2. 当第 $n - 1$ 个数为 $1$ 时：

      此时第 $n - 2$ 个数必为 $0$ , 否则将不合法，也即最后三个数字必是 “$011$”, 那么此时只要保证前 $n - 3$ 个数合法即可。合法的数量为 $f(n - 3)$ 。 

此时就求出了递推式： $f(n) = f(n - 1) + f(n - 2) + f(n - 3)$ 。结合 $f(1) = 1, f(2) = 4, f(3) = 7$ 即可线性递推出 $f(n)$ 的值。

### 参考代码

```C++
#include <bits/stdc++.h>
using namespace std;
const int mod = 1e9 + 7;

signed main() {
    int n;
    cin >> n;
    vector<int> f(n + 2);
    f[0] = 1, f[1] = 2, f[2] = 4;
    for (int i = 3; i <= n; i++) f[i] = ((f[i - 1] + f[i - 2]) % mod + f[i - 3]) % mod;
    cout << f[n] << endl;
    return 0;
}
```
