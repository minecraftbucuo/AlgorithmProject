# 冒险 题解

分类讨论即可

**先讨论明显不可行的方案**  

$n>m$时 显然不可以,通过操作n只会变大,不可能到达m

$n+1=m$ 显然不可以,一次操作至少增加2,不可到达m

**讨论可行的方案**  
先来证明在$n<=m\And m \ne n+1$的情况一定有解
显然$n=m$的情况答案为0

那么对于其他情况  
设 $d=m-n$ 则 $d>=2$  
实际上我们是去看是否只用$2,3$两种数字拼出d,即$k_1*2+k_2*3 =d$
我们的答案是让  $k_1+k_2$ 最小  
我们贪心地想尽可能的多用大的数3,来使答案更小

对此,
我们对 $d$对3的余数进行分类讨论  

1. $d\%3=0$ 此时$k_1=0,k_2=\frac{d}{3}$ 显然此时答案存在且是最优
2. $d\%3=2$ 此时$k_1=1,k_2=\lfloor\frac{d}{3}\rfloor$ 显然此时答案存在且是最优,如果不能发现,可以尝试将2个3替换成3个2(保证和还是d),可以发现答案一定会更差
3. $d\%3=1$ 此时$k_1=2,k_2=\lfloor\frac{d}{3}\rfloor-1$ 验证方式最优方式同上


```c++
#include<bits/stdc++.h>
using namespace std;


int main()
{
    int T;cin>>T;
    while (T--)
    {
        int n,m;cin>>n>>m;
        if (m<n||m==n+1)cout<<"QAQ"<<endl;
        else
        {
            int d=m-n;
            int ans;
            if (d%3==0)ans=d/3;
            else ans=d/3+1;
            cout<<ans<<endl;
        }
    }
    return 0;
}
